I just had this problem too with both the main HEAD and 0.9.0 tag on a fresh install on MacOS.  
The problem is that pyproject.toml specifies a git repo instead of the 2.0.12 that is in the poetry.lock for swebench:

pyproject.toml 
```
114   [tool.poetry.group.evaluation.dependencies]
...
119 - swebench = { git = "https://github.com/All-Hands-AI/SWE-bench.git" }
119 + swebench = "*" # { git = "https://github.com/All-Hands-AI/SWE-bench.git" }
```
